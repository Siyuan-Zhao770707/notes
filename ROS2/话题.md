
**话题：节点间传递数据的桥梁**
	1. 发布/订阅模型、
	2. 订阅者和发布者不唯一，多对多
	3. 异步通信机制（发布者不知道订阅者什么时候收到数据）
	4. 话题适合周期发布的数据，不适合逻辑较强的数据
	5. msg文件定义通信的消息结构（统一数据的描述格式）

#### 实现简单的话题通信
在[[节点]]基础上进行功能添加

**publisher**：
	1. 程序入口初始化
	2. 创建节点并且初始化
	3. 创建发布者对象
	4. 创建并填充话题消息
	5. 发布话题消息
	6. 销毁节点并关闭窗口
*learning_topic -> learning_topic -> topic_helloworld_pub.py*
```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String   # ROS2 标准字符串消息类型


class PublisherNode(Node):

    def __init__(self, name):
        super().__init__(name)
        # 创建发布者（消息类型，话题名，队列长度）
        # 队列长度：消息发布不及时时的缓存上限
        # 超出时丢弃最旧的消息，存放最新的
        self.pub = self.create_publisher(String, "chatter", 10)
        # 创建定时器，每 0.5 秒执行一次回调
        self.timer = self.create_timer(0.5, self.timer_callback)

    def timer_callback(self):
        """定时发布消息"""
        msg = String()
        msg.data = 'Hello World'
        self.pub.publish(msg)                        # 发布到话题
        self.get_logger().info('Publishing: "%s"' % msg.data)


def main(args=None):
    rclpy.init(args=args)
    node = PublisherNode("topic_helloworld_pub")
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()
```

**subscriber**:
	1. 编程接口初始化
	2. 创建节点并初始化
	3. 创建订阅者对象
	4. 回调函数处理话题数据
	5. 销毁节点并关闭窗口
*learning_topic -> learning_topic -> topic_helloworld_sub.py*
```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String


class SubscriberNode(Node):

    def __init__(self, name):
        super().__init__(name)
        # 创建订阅者（消息类型，话题名，回调函数，队列长度）
        self.sub = self.create_subscription(
            String, "chatter", self.listener_callback, 10)

    def listener_callback(self, msg):
        """收到话题消息时打印内容"""
        self.get_logger().info('I heard: "%s"' % msg.data)


def main(args=None):
    rclpy.init(args=args)
    node = SubscriberNode("topic_helloworld_sub")
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()
```
