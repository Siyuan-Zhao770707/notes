  
  **工作空间**：存放项目开发文件的文件夹 
  典型的工作空间结构：
  *dev_ws*
	  1. *src*：代码空间(Source Space)**编写代码的地方**
	  2. *install*：安装空间(Intall Space)**可执行文件和脚本**
	  3. *build*：编译空间(Build Space)**编译过程的中间文件**
	  4. *log*：日志空间(Log Space )**运行过程中的各种警告、错误的记录**

以下的内容步骤在日常使用的时候可以使用这些shell脚本来代替
[[ros2 常用shell脚本]]
#### 初始化环境（只用做一次）

``` zsh
#下载rosdepc（国内镜像，自动安装ROS包所需要的依赖）
sudo pipx install rosdepc

# 创建工作空间
mkdir -p ~/dev_ws/src
#进入工作空间的代码空间
cd dev_ws/src

#初始化并且更新
#初始化后仍然有可能会有依赖没有安装，到时候再下载也行
sudo rosdepc init
sudo rosdepc update

```

#### 编译代码
对着空的src文件夹编译就能够初始化

``` zsh
#先要有编译工具
#安装编译器工具colcon
sudo apt install python3-colcon-ros

#进入根目录，colcon编译一定要再工作空间的根目录下
cd dev_ws

#使用colcon编译代码(不过目前代码空间还没有代码)
colcon build

# 推荐使用这个方式编译，能够在调试的时候省下空间
colcon build --symlink-install

# 在正式交付的时候使用这个方式编译，能够省去调试模式的代码库，进一步压缩空间
colcon build --symlink-install --cmake-args -DCMAKE_BUILD_TYPE = Release

# 如果不注意很容易把磁盘占满，定时要清理一下编译文件重新编译
``` 

#### 配置环境变量（一定要在编译之后才能做）

```zsh
#在运行的时候让系统能够认识你的功能包，每次进入都需要敲一次
source ～/dev_ws/install/local_setup.sh 

#可以将这个指令放入.bashrc(.zshrc)里面就不用每次都要敲这个指令了
vim ~/.zshrc
#在文件的最后一行添加
source ～/dev_ws/install/local_setup.sh
#在里面添加那一行然后:wq保存退出
source ~/.zshrc
```


**功能包**：能够实现不同的功能，方便分享，降低耦合关系，提高软件复用率

#### 创建功能包

```zsh
#进入代码空间
cd dev_ws/src/
#创建包
ros2 pkg create --build-type <包类型> <包的名字>
ros2 pkg create --build-type ament_cmake learning_pkg_c #C++
ros2 pkg create --build-type ament_python learning_pkg_python #python
```

#### 编译功能包

```zsh
# 进入工作空间根目录
cd dew_ws/
# 编译包
colcon build # 在你每次修改或者添加新的代码文件的时候需要重新运行一次
source install/local_setup.bash # 在添加了新的功能包的时候需要重新运行一次
```
